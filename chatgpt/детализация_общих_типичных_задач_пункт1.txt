Конечно! Ниже представлены примеры задач для первого пункта «Разработка конфигураций и доработка типовых решений», которые можно использовать при создании бенчмарка AI-инструментов для разработчиков 1С
1. Создание новых модулей

Задача 1 Разработка модуля управления персональными скидками для клиентов

    Описание Создать подсистему, позволяющую устанавливать индивидуальные скидки для клиентов на определенные товары или группы товаров с учетом различных условий (срок действия, объем покупки и т.д.).
    Требования
        Реализовать справочник «Персональные скидки» с возможностью настройки условий применения.
        Модифицировать документы продажи для автоматического применения соответствующих скидок.
        Создать отчет «Анализ предоставленных персональных скидок».

Задача 2 Создание модуля уведомлений для пользователей системы

    Описание Разработать механизм отправки уведомлений пользователям о важных событиях (например, о приближении сроков платежей, необходимости обновления данных и т.д.).
    Требования
        Реализовать обработку для настройки типов уведомлений и каналов доставки (внутренние сообщения, email, SMS).
        Интегрировать модуль с существующими бизнес-процессами для автоматической генерации уведомлений.
        Обеспечить возможность отслеживания истории отправленных уведомлений.

2. Доработка существующих конфигураций

Задача 3 Модификация конфигурации «1СУправление торговлей» для учета серийных номеров товаров

    Описание Добавить функциональность для отслеживания серийных номеров товаров на всех этапах движения от поступления на склад до продажи клиенту.
    Требования
        Добавить в документы поступления и реализации товаров возможность указания серийных номеров.
        Настроить складской учет для отслеживания остатков по серийным номерам.
        Создать отчет «Движение товаров по серийным номерам».

Задача 4 Доработка конфигурации «1СБухгалтерия» для автоматизации расчета налога на имущество

    Описание Автоматизировать процесс расчета налога на имущество организации с учетом изменений в законодательстве.
    Требования
        Обновить справочник основных средств, добавив необходимые реквизиты для расчета налога.
        Реализовать обработку для автоматического расчета налога на основании данных о стоимости и амортизации основных средств.
        Обновить регламентированную отчетность для отражения рассчитанных сумм налога.

3. Настройка прав доступа

Задача 5 Настройка разграничения доступа в системе для новых подразделений компании

    Описание В связи с расширением компании необходимо настроить права доступа для сотрудников новых подразделений, обеспечив им доступ только к соответствующим данным.
    Требования
        Создать новые роли для сотрудников каждого подразделения с учетом их должностных обязанностей.
        Настроить ограничения на уровне записей (RLS) для доступа только к данным своего подразделения.
        Провести тестирование настроек прав доступа с разными учетными записями.

Задача 6 Реализация механизма двухфакторной аутентификации для повышения безопасности

    Описание Усилить безопасность системы путем внедрения двухфакторной аутентификации при входе пользователей.
    Требования
        Интегрировать систему с сервисом отправки SMS или использованием мобильного приложения для генерации кодов.
        Настроить дополнительную проверку при аутентификации пользователей.
        Обеспечить возможность восстановления доступа при утере второго фактора.

Применение этих задач в бенчмарке позволит оценить эффективность AI-инструментов в следующих аспектах

    Генерация кода и предложений Насколько точно и релевантно AI-инструмент предлагает решения по реализации заданной функциональности.
    Соответствие стандартам разработки Проверка, соответствует ли сгенерированный код лучшим практикам и стандартам разработки на платформе 1С.
    Скорость разработки Сравнение времени, затраченного на выполнение задачи с использованием AI-инструмента, с традиционным подходом.
    Качество и надежность Оценка количества ошибок и проблем, выявленных при тестировании предложенных AI-решений.

Пример оценки AI-инструмента на одной из задач

    Задача Модификация конфигурации для учета серийных номеров товаров.
    Критерии оценки
        Полнота решения AI-инструмент должен предложить полную реализацию функционала, включая изменения в документах, справочниках и отчетах.
        Качество кода Код должен быть оптимизированным, читаемым и поддерживаемым.
        Интеграция с существующей системой Решение не должно нарушать работу текущих механизмов и должно быть совместимо с обновлениями конфигурации.
        Тестирование Минимальное количество ошибок при тестировании функционала.

Заключение

Предложенные задачи отражают реальные сценарии, с которыми сталкиваются разработчики 1С, и могут служить основой для создания объективного и релевантного бенчмарка AI-инструментов. Это поможет оценить их практическую пользу и влияние на эффективность процесса разработки.
