Задачи для пункта 5: Миграция данных (в упрощенном виде)

Чтобы быстро запустить первую версию вашего бенчмарка, предлагаю разработать несколько простых и типичных задач по миграции данных, которые часто встречаются в работе разработчиков 1С. Эти задачи помогут оценить, насколько AI-инструменты могут помочь в написании скриптов для переноса и преобразования данных.
Задача 1: Перенос данных из одного справочника в другой с изменением структуры данных

    Описание задачи: У вас есть два справочника: "СтарыйСправочникТоваров" с полями "КодТовара", "НазваниеТовара", "Цена" и "Количество", и "НовыйСправочникТоваров" с полями "Артикул", "Наименование", "ЦенаЗакупки", "ЦенаПродажи". Необходимо перенести данные из старого справочника в новый, при этом:
        "КодТовара" соответствует "Артикул".
        "НазваниеТовара" соответствует "Наименование".
        "Цена" соответствует "ЦенаЗакупки".
        "ЦенаПродажи" рассчитывается как "ЦенаЗакупки" * 1.2.
        Поле "Количество" не переносится.

    Ожидаемый результат: Скрипт, который корректно переносит и преобразует данные, создавая новые записи в "НовыйСправочникТоваров" с учетом указанных соответствий и расчетов.

Задача 2: Объединение данных из нескольких справочников в один

    Описание задачи: Имеются два справочника: "СправочникПоставщиков" и "СправочникПокупателей", оба имеют поля "Название", "ИНН", "Телефон". Нужно объединить эти данные в единый справочник "Контрагенты", добавив поле "ТипКонтрагента" со значениями "Поставщик" или "Покупатель" соответственно.

    Ожидаемый результат: Скрипт, который создает записи в справочнике "Контрагенты", объединяя данные из обоих справочников и устанавливая правильный тип контрагента.

Задача 3: Преобразование формата данных в существующих записях

    Описание задачи: В документе "Продажи" поле "Сумма" хранится в виде строки (например, "1000.50"). Необходимо преобразовать это поле в числовой формат с двумя знаками после запятой для всех существующих записей.

    Ожидаемый результат: Скрипт, который проходит по всем записям документа "Продажи" и преобразует значение поля "Сумма" в числовой формат Decimal(15, 2).

Задача 4: Заполнение нового поля на основе существующих данных

    Описание задачи: В справочник "Сотрудники" добавлено новое поле "ПолноеИмя", которое должно содержать объединение полей "Фамилия", "Имя" и "Отчество". Нужно заполнить это поле для всех существующих записей.

    Ожидаемый результат: Скрипт, который обновляет каждую запись в справочнике "Сотрудники", заполняя поле "ПолноеИмя" в формате "Фамилия Имя Отчество".

Задача 5: Удаление дубликатов записей по определенному критерию

    Описание задачи: В справочнике "Клиенты" есть дублирующиеся записи, где совпадают значения полей "ИНН" и "КПП". Необходимо удалить дубликаты, сохранив одну запись и перенести все связанные документы к ней.

    Ожидаемый результат: Скрипт, который находит дубликаты по "ИНН" и "КПП", удаляет лишние записи и перенаправляет ссылки в связанных документах на оставшуюся запись.

Задача 6: Импорт данных из внешнего файла

    Описание задачи: У вас есть CSV-файл с информацией о товарах: "Артикул;Наименование;Цена". Нужно импортировать эти данные в справочник "Номенклатура", при этом если товар с таким "Артикулом" уже существует, обновить его "Цена" и "Наименование".

    Ожидаемый результат: Скрипт или обработка, которая считывает данные из CSV-файла и корректно обновляет или создает записи в справочнике "Номенклатура".

Задача 7: Массовое обновление данных по условию

    Описание задачи: В справочнике "Номенклатура" необходимо увеличить цену на 5% для всех товаров, у которых "Категория" равна "Продукты питания".

    Ожидаемый результат: Скрипт, который выборочно обновляет поле "Цена" для указанных товаров, увеличивая значение на 5%.

Задача 8: Перенос данных между регистрами сведений

    Описание задачи: Существует регистр сведений "ЦеныПоставщиков" с измерениями "Товар", "Поставщик" и ресурсом "Цена". Нужно перенести данные в новый регистр "ЦеныКонтрагентов", добавив измерение "Дата", которое будет установлено текущей датой для всех записей.

    Ожидаемый результат: Скрипт, который переносит данные из одного регистра в другой, корректно заполняя новое измерение.

Задача 9: Архивирование старых документов

    Описание задачи: Документы "ПоступлениеТоваров" датированные ранее 01.01.2020 должны быть перемещены в архивную информационную базу или помечены на удаление, чтобы облегчить основную базу.

    Ожидаемый результат: Скрипт, который находит такие документы и либо экспортирует их в файл для дальнейшего импорта в архивную базу, либо помечает на удаление.

Задача 10: Консолидация справочников из разных баз

    Описание задачи: Имеются две базы данных с одинаковыми структурами справочника "Сотрудники". Нужно объединить данные этих справочников в одной базе, при этом избежать дублирования сотрудников с одинаковым "ТабельнымНомером".

    Ожидаемый результат: Скрипт или обработка, которая импортирует данные из одной базы в другую, сопоставляя записи по "ТабельномуНомеру" и объединяя информацию.

Как использовать эти задачи в вашем бенчмарке:

    Подготовка данных:
        Создайте тестовые данные в соответствии с описанием каждой задачи.
        Убедитесь, что данные достаточно просты для быстрого тестирования, но содержат элементы, необходимые для оценки возможностей AI-инструмента.

    Разработка сценариев тестирования:
        Опишите каждую задачу в виде четкого задания для AI-инструмента.
        Определите, какую помощь ожидаете от инструмента (например, генерация кода, предложений по преобразованию данных и т.д.).

    Критерии оценки:
        Время выполнения: Сколько времени требуется инструменту для предоставления решения.
        Качество решения: Насколько корректен и оптимален сгенерированный код.
        Требуемые доработки: Объем исправлений, которые должен внести разработчик после использования инструмента.
        Удобство использования: Насколько понятен и удобен интерфейс и ответы инструмента.

    Проведение тестирования:
        Используйте каждый AI-инструмент для решения всех задач.
        Записывайте полученные результаты и наблюдения.

    Анализ результатов:
        Сравните эффективность инструментов по каждому из критериев.
        Определите сильные и слабые стороны каждого инструмента в контексте задач миграции данных.

    Отчетность:
        Составьте отчет с выводами и рекомендациями на основе проведенного тестирования.
        Представьте результаты в удобной для восприятия форме (таблицы, графики, диаграммы).

Заключение:

Эти задачи помогут вам быстро запустить первую версию бенчмарка, сосредоточившись на типичных и часто встречающихся задачах миграции данных в 1С. Они достаточно просты для быстрой реализации, но при этом предоставят ценную информацию о возможностях AI-инструментов в реальных сценариях разработки.
